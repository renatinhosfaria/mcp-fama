# Meta v25 Fixes Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Corrigir o MCP para a API Meta v25.0 em criação de campanha, tratamento de erro `code=100` e filtros de status de listagem.

**Architecture:** O ajuste fica concentrado em schemas e helpers já existentes. Vamos cobrir o comportamento com testes de `node:test`, depois alterar o payload de criação de campanha, o enum usado nos filtros de listagem e a priorização de `error_user_msg` no formatter de erros da Meta.

**Tech Stack:** TypeScript, Zod, node:test, tsx

---

### Task 1: Lock The Regressions With Tests

**Files:**
- Create: `tests/meta-v25-fixes.test.ts`

**Step 1: Write the failing tests**

Cobrir:
- `meta_create_campaign` deve expor `is_adset_budget_sharing_enabled` com default `false`
- o payload de criação deve carregar esse campo
- `handleApiError` deve usar `error_user_msg` para `code=100`
- `meta_list_campaigns`, `meta_list_adsets` e `meta_list_ads` não devem aceitar `DELETED` em `status_filter`

**Step 2: Run test to verify it fails**

Run: `node --import tsx --test tests/meta-v25-fixes.test.ts`
Expected: FAIL pelos comportamentos ainda ausentes.
